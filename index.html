Let's Get Started

Introduction

This tutorial will help you get started using Clique, a component of the Resonant data analysis and visualization platform that allows you to explore and visualize large graphs and networks interactively in your browser.

In this tutorial, we will visualize a subset of the Facebook friendship graph, allowing us to start to explore the social dynamics via connectivity in this graph. To create this interactive visualization, we will develop a minimalistic web application that uses Clique to manage importation of graph data from a database, d3 and WebCola to render the graph in a familiar, interactive fasion on screen, and standard HTML and JavaScript to create an example of UI control. By the end of this tutorial, you should have a basic grasp of how to construct an application with Clique, and hopefully some ideas of where to go next.

Let's get started!

Set Up an Environment and Get Some Data

We will be constructing a web application piece by piece, so we'll need a clean environment in which to work. Create a new directory somewhere, and move into it. The first thing we will do is download a graph of anonymized Facebook users and their friendship network. facebook-sample-data.txt is a simple, tab-separated data file containing rows of two numbers: an ID for a user, and an ID for another user found in the first user's friend list. Download this file so it's available for use by the web application.

Now let's create a basic infrastructure for a web application.

The call to d3.tsv.parseRows() (line 4 ff.) splits the raw data into an array of arrays containing the tab-separated field values from each line, then applies the function specified to each row. Our function simply converts the strings into numbers, preserving the array structure.

After that, we create a list of nodes and links to be passed to the Graph model in the next step. This is an object with two fields, nodes and links. nodes contains a list of objects encoding nodes with metadata. In this case, we are installing the numeric identifier of each node in the data file as its "id" metadata field. links contains a list of objects with "source" and "target" fields, which reference entries in the nodes list by index.

This structure is known as a "node link list", and will be used in the next step to construct a Graph model.

First, line 44 creates an adapter - an object that takes data in some form and shapes it into the form needed by the Graph model, which is created in line 47. Together, these two objects work to provide information about the graph to other components of the application.

The most important such component is the view, created in line 53. Note the el (element) property on line 55; this specifies where the view should be inserted in the web page. In our case "#graph" is a CSS selector that references the SVG element in our HTML file.

Note also the fill option property (line 56), which uses a Category-10 colormap supplied by D3 to color code the nodes by the identity of the last digit of their IDs. By modifiying this function, you can colormap by other data qualities that matter more to your particular application.

The remainder of the function specifies a few things for convenience. For now, the important ones are in lines 76 and 88, respectively.

The addNode() function takes in an integer argument and asks the graph adapter to find a node with that ID, and add it (if it exists) to the graph view. Try it out right now - open the developer console, and type in addNode(1) and hit enter. You should see a single node appear.

Line 61 uses the view's built-in selection model to enable the application to react to nodes and links being clicked on. In this case, the application will print out a node's ID on the console whenever it is selected via click. You can verify the identity of the new node you have just added by clicking once on the node.

Next, if you want to see the neighbors of this node, you can use the expandNode() function (line 88) to bring in its neighbors. expandNode(), like addNode(), takes the ID of a node that has already been added, and updates the graph appropriately. Try it now - type expandNode(1) and hit enter.

You can imagine that it may become burdensome to flip back and forth between the view and the console, typing out repetitive commands, just to explore the graph. Line 67 solves this problem by attaching an action to the double click event on nodes. This line states that whenever the view renders, we want to attach a double click event on each node, which simply automates the call to expandNode() that we just experimented with manually. The callback operates on a D3 selection, and thus takes a parameter d representing the data carried by the node, including its ID. Now, whenever a node is double clicked, it will expand exactly as though you had expanded it by name using the manual function in the developer console.

This is meant to demonstrate how your application can respond to UI events such as clicks, double clicks, drags, etc. Another approach would be to create standard UI elements in the application, and attach such actions to those instead.

At this point, you can click around, explore the graph, and begin to understand the structure of the Facebook friendships it represents.
